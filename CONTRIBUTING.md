# 贡献指南 | Contributing Guidelines

欢迎贡献！本项目采用开放协作模式。

## 🤝 如何贡献

### 1. 报告问题

发现错误或有问题？请创建 [Issue](https://github.com/yrju/long-term-memory-research/issues)。

**报告问题时请提供**:
- 问题描述
- 复现步骤
- 预期行为
- 实际行为
- 截图 (如适用)

### 2. 提交内容

可以贡献以下内容:

#### 文档类
- 📝 新论文摘要
- 🏢 工业案例
- 🔧 开源项目更新
- 📊 性能基准测试
- 🌍 文档翻译

#### 代码类
- 🐛 Bug 修复
- ✨ 新功能
- 📦 代码示例
- 🧪 测试用例
- 📚 文档注释

#### 改进类
- 💡 改进建议
- 🎨 格式优化
- 🔤 拼写纠正
- 🔗 链接修复

### 3. 提交流程

```bash
# 1. Fork 项目
# 在 GitHub 上点击 Fork 按钮

# 2. 克隆到本地
git clone https://github.com/YOUR_USERNAME/long-term-memory-research.git
cd long-term-memory-research

# 3. 创建分支
git checkout -b feature/your-feature-name

# 4. 进行修改
# 编辑文件...

# 5. 提交更改
git add .
git commit -m "Add: your feature description"

# 6. 推送到远程
git push origin feature/your-feature-name

# 7. 创建 Pull Request
# 在 GitHub 上创建 PR
```

## 📋 代码规范

### 文档格式

- 使用 Markdown 格式
- 标题层级清晰 (H1 → H2 → H3)
- 代码块标注语言
- 链接使用相对路径 (内部) 或完整 URL (外部)

### 提交信息

遵循以下格式:

```
<type>: <subject>

<body>

<footer>
```

**Type 类型**:
- `Add`: 新增内容
- `Update`: 更新内容
- `Fix`: 修复错误
- `Remove`: 删除内容
- `Refactor`: 重构代码
- `Docs`: 文档更新
- `Style`: 格式调整

**示例**:
```
Add: 新的工业案例 - 某电商平台

- 添加电商客服案例
- 包含效果指标和成本分析
- 参考 Zendesk 案例格式

Closes #123
```

### 文档结构

新增文档时遵循现有结构:

```markdown
# 标题

**最后更新**: YYYY-MM-DD
**成熟度**: ⭐⭐⭐⭐⭐
**推荐指数**: 🔥 强烈推荐

---

## 1. 概述

...

## 2. 核心内容

...

## 3. 实施方案

...

## 4. 最佳实践

...

## 5. 参考资料

...
```

## 🔍 审核流程

1. **自动检查**: GitHub Actions 运行格式检查
2. **人工审核**: 维护者审核内容和质量
3. **合并**: 审核通过后合并到主分支

### 审核标准

- ✅ 内容准确
- ✅ 格式规范
- ✅ 来源可靠
- ✅ 语言清晰
- ✅ 无版权问题

## 💬 讨论

有疑问或想法？欢迎在 [Discussions](https://github.com/yrju/long-term-memory-research/discussions) 讨论。

### 讨论分类

- 💡 Ideas: 新功能建议
- 📚 Documentation: 文档改进
- 🤔 Q&A: 问题咨询
- 📢 Announcements: 公告
