### 2. 记忆系统配置 (openclaw.json)

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **enabled** | `true` | 启用记忆系统 |
| **sources** | `["memory", "sessions"]` | 记忆源：memory 目录 + 会话转录 |
| **provider** | `auto` | 自动选择嵌入提供商 |
| **vector.enabled** | `true` | 启用向量检索 |
| **sync.onSessionStart** | `true` | 会话启动时同步 |
| **sync.watch** | `true` | 文件监听自动同步 |
| **sync.intervalMinutes** | `30` | 每 30 分钟定期同步 |
| **query.maxResults** | `5` | 最多返回 5 条结果 |
| **query.hybrid.enabled** | `true` | 启用混合检索 (FTS + Vector) |
| **query.hybrid.vectorWeight** | `0.7` | 向量权重 70% |
| **query.hybrid.textWeight** | `0.3` | 文本权重 30% |
| **query.hybrid.mmr.enabled** | `true` | 启用 MMR 去重 |
| **cache.enabled** | `true` | 启用查询缓存 |

## 📞 问题排查

### 常见问题

**Q1: 推送失败 "Repository not found"**
- A: 需要先在 GitHub 上创建仓库

**Q2: 记忆系统未生效**
- A: 重启 Gateway：`openclaw gateway restart`

**Q3: Session Memory Hook 未触发**
- A: 检查配置 `hooks.internal.entries.session-memory.enabled` 是否为 `true`

**Q4: 向量检索不可用**
- A: 检查 SQLite 向量扩展是否加载，查看 `openclaw memory status` 输出
