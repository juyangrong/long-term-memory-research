# 03-NL2DSL 方案设计

## 📖 概述

本文档详细描述自然语言到 DSL（领域特定语言）的转换方案设计，包括意图识别、实体抽取、语义链接和 DSL 生成等核心模块。

---

## 🎯 设计目标

1. **高准确率** - 语义解析准确率 > 85%
2. **强可解释性** - 每个解析步骤都可追溯和验证
3. **低延迟** - 端到端解析时间 < 500ms
4. **易扩展** - 支持新业务语义快速接入

---

## 🔧 核心模块设计

### 1. 预处理模块

**功能：**
- 中文分词（jieba/HanLP）
- 拼写错误检测和校正
- 同义词标准化
- 停用词过滤

**示例：**
```
输入："查一下上个季度的销售额在华东地区的表现"
处理后：["查询", "上季度", "销售额", "华东地区", "表现"]
```

### 2. 意图识别模块

**查询类型分类：**
- **对比类** - 比较不同维度下的指标
- **趋势类** - 时间序列分析
- **分布类** - 数据分布统计
- **明细类** - 原始数据查询
- **聚合类** - 汇总统计

**业务域识别：**
- 交易域
- 用户域
- 产品域
- 营销域
- 财务域

**技术实现：**
- 基于 BERT 的多分类模型
- 支持 few-shot learning
- 置信度阈值 > 0.7

### 3. 实体抽取模块

**实体类型：**
- **术语 (Term)** - 业务词汇
- **指标 (Metric)** - 可量化度量
- **维度 (Dimension)** - 分析角度
- **时间 (Time)** - 时间范围
- **数值 (Number)** - 过滤条件值

**抽取方法：**
- BERT-BiLSTM-CRF 模型
- 规则引擎补充
- 语义层词典匹配

**实体链接：**
```
抽取实体："销售额"
  ↓
语义层匹配：
  - TERM_001: GMV (商品交易总额)
  - TERM_002: Revenue (营业收入)
  ↓
置信度评分：TERM_001 (0.92), TERM_002 (0.15)
  ↓
链接结果：TERM_001 (GMV)
```

### 4. 语义校验模块

**校验规则：**
1. **类型校验** - 实体类型与意图匹配
2. **关系校验** - 实体间关系在语义层存在
3. **逻辑校验** - 查询条件无矛盾
4. **权限校验** - 用户有数据访问权限

**校验流程：**
```
DSL 草稿
  ↓
语义层查询
  ↓
规则引擎校验
  ↓
通过/失败 + 错误信息
```

### 5. DSL 生成模块

**DSL 语法设计：**
```
SELECT <metrics>
[WHERE <conditions>]
[GROUP BY <dimensions>]
[ORDER BY <metric> ASC|DESC]
[LIMIT <n>]
```

**生成策略：**
- 基于模板的填充
- 语法树构建
- 规范化输出

**示例：**
```
输入："查看华东地区上个季度的销售额和利润"
输出：
SELECT 指标 (销售额，利润)
WHERE 维度 (地区，=，华东)
  AND 维度 (时间，BETWEEN，2024-Q1)
GROUP BY 维度 (地区)
```

## 📊 技术方案对比

### 方案 1: 纯规则引擎

**优势：**
- 可解释性强
- 维护成本低
- 确定性高

**劣势：**
- 覆盖率有限
- 泛化能力弱
- 规则冲突难处理

**适用场景：** 业务简单、查询模式固定

### 方案 2: 纯 LLM

**优势：**
- 泛化能力强
- 覆盖率高
- 开发成本低

**劣势：**
- 可解释性弱
- 幻觉问题
- 难以约束

**适用场景：** 探索性场景、容忍一定错误率

### 方案 3: LLM+ 规则混合 ⭐ 推荐

**优势：**
- 兼顾泛化和可解释性
- 规则约束减少幻觉
- LLM 处理长尾场景

**劣势：**
- 架构复杂度高
- 需要调优平衡点

**适用场景：** 企业级生产环境

## 🔍 错误处理机制

### 1. 置信度阈值

```
IF 意图置信度 < 0.7:
  → 请求用户澄清
  → 提供候选意图列表

IF 实体链接置信度 < 0.6:
  → 使用语义相似度推荐
  → 用户确认机制
```

### 2. 降级策略

```
完整解析失败
  ↓
规则匹配（降级 1）
  ↓
关键词检索（降级 2）
  ↓
返回错误提示 + 建议
```

### 3. 反馈闭环

```
用户查询 → 解析结果 → 用户反馈
                              ↓
                        标注数据积累
                              ↓
                        模型迭代优化
```
