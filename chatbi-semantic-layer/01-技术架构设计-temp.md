# 01-技术架构设计

## 📖 概述

本文档描述 ChatBI 语义层的整体技术架构设计，包括模块划分、数据流向、技术选型等核心内容。

---

## 🏗️ 架构设计原则

### 1. 分层解耦
- **应用层** - 面向用户的交互界面
- **语义层** - 业务语义的统一表达和管理
- **转换层** - 自然语言到查询语句的转换引擎
- **数据层** - 底层数据存储和计算引擎

### 2. 可扩展性
- 插件化架构设计
- 支持新数据源快速接入
- 支持新业务语义无缝扩展

### 3. 高性能
- 查询响应时间 < 3 秒
- 语义解析准确率 > 90%
- 支持高并发查询

### 4. 可维护性
- 完整的日志和监控体系
- 清晰的模块边界和接口定义
- 完善的文档和测试覆盖

---

## 📐 系统架构

### Layer 4: 应用层
- ChatBI 界面 (Web/Mobile)
- 报表/看板 (Dashboard)
- 数据 API (REST/GraphQL)

### Layer 3: 语义层
- GraphRAG 语义知识图谱
  - 术语管理 (Term Service)
  - 指标管理 (Metric Service)
  - 维度管理 (Dimension Service)
- 语义, 语义关系图谱 (Graph Database)

### Layer 2: 转换层
- NL2DSL  引擎
- DSL2SQL 引擎

### Layer 1: 数据层
- 数据仓库 (Hive/ODPS)
- 数据湖 (Iceberg)
- 实时数仓 (Kafka/DS)
- 血缘解析引擎

---

## 🔧 核心模块设计

### 1. 术语管理
- 业务术语的 CRUD 操作
- 术语同义词管理
- 术语版本控制

### 2. 指标管理
- 指标定义和计算公式
- 指标与术语关联
- 指标血缘追踪

### 3. 维度管理
- 维度层级关系维护
- 维度值域管理

### 4. NL2DSL 引擎
- 意图识别
- 实体抽取
- 语义链接
- DSL 生成

### 5. DSL2SQL 引擎
- DSL 解析
- 语义校验
- 查询优化
- SQL 生成

---

## 📊 数据流向

```
用户 NL 查询 → NL2DSL 解析 → 语义层校验 → DSL2SQL 转换 → SQL 执行 → 结果返回
```
