# 02-语义层建设方案

## 1. 语义层概述

### 1.1 什么是语义层

语义层 (Semantic Layer) 是介于业务用户和底层数据之间的抽象层，它将技术性的数据结构转换为业务用户能够理解的概念模型。

**核心价值：**
- 📌 **统一语言** - 消除业务与技术之间的术语歧义
- 📌 **自助分析** - 业务人员无需 SQL 技能即可查询数据
- 📌 **一致性保证** - 相同指标在不同场景下计算逻辑一致
- 📌 **可追溯性** - 每个数据点都能追溯到源头

### 1.2 语义层 vs 传统指标平台

| 维度 | 传统指标平台 | 语义层 (本方案) |
|------|-------------|----------------|
| 核心能力 | 指标定义与计算 | 语义理解与转换 |
| 使用方式 | 配置化界面 | 自然语言交互 |
| 技术实现 | 规则引擎 | GraphRAG + LLM |
| 灵活性 | 低 (需预定义) | 高 (动态解析) |
| 准确率 | 高 (确定性) | 中高 (需持续优化) |

## 3. GraphRAG 语义图谱构建

### 3.1 图谱 Schema 设计

```
(Term)-[RELATED_TO]->(Term)
(Term)-[HAS_METRIC]->(Metric)
(Term)-[HAS_DIMENSION]->(Dimension)
(Metric)-[CALCULATED_FROM]->(Table/Column)
(Metric)-[HAS_DIMENSION]->(Dimension)
(Dimension)-[HAS_HIERARCHY]->(Dimension)
(Table)-[DERIVED_FROM]->(Table)  // 血缘关系
```

### 3.2 图谱构建流程

```
Step 1: 数据源接入
  ├── 元数据同步 (DataCloud)
  ├── 血缘数据同步
  └── 现有指标系统对接

Step 2: 语义抽取
  ├── 自动抽取 (基于规则和 NLP)
  │   ├── 表名/字段名 → 候选术语
  │   ├── 注释信息 → 候选定义
  │   └── 血缘关系 → 候选关联
  └── 人工补充 (业务专家)
      ├── 术语标准化
      ├── 指标定义确认
      └── 维度层级梳理

Step 3: 图谱构建
  ├── 实体创建
  ├── 关系建立
  └── 属性填充

Step 4: 质量校验
  ├── 完整性检查
  ├── 一致性检查
  └── 准确性验证

Step 5: 发布上线
  ├── 版本管理
  ├── 变更通知
  └── API 暴露
```

### 3.3 图谱存储方案

| 存储类型 | 技术选型 | 存储内容 |
|---------|---------|---------|
| 图数据库 | Neo4j / Nebula Graph | 实体关系网络 |
| 向量数据库 | Milvus / Weaviate | 语义向量索引 |
| 关系数据库 | MySQL / PostgreSQL | 实体属性数据 |
| 缓存 | Redis | 热点语义数据 |

## 4. 语义检索与增强

### 4.1 检索策略

**混合检索模式：**
```
1. 关键词检索 (BM25) - 精确匹配术语名称
2. 向量检索 (Embedding) - 语义相似度匹配
3. 图遍历检索 - 关系路径查找
4. 融合排序 - 综合得分排名
```

### 4.2 RAG 增强流程

```
用户查询 → 检索相关语义 → 构建上下文 → LLM 解析 → 生成 DSL
    │            │              │           │         │
    │            │              │           │         └→ 校验修正
    │            │              │           └→ 置信度评估
    │            │              └→ Prompt 构建
    │            └→ TopK 语义单元
    └→ 意图识别
```
