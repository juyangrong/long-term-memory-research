# 07-技术选型与竞品分析

本文档对国内外主流语义层和 NL2SQL 技术方案进行深入调研和对比分析，为 ChatBI 语义层的技术选型提供决策依据。

## 🌍 行业背景

### 语义层发展历史

```
1990s: OLAP 时代
  └── Business Objects 提出"语义层"概念
  └── 目标：让业务用户理解数据结构
  
2000s: BI 工具时代
  └── Tableau、QlikView 可视化探索
  └── 仍需要技术人员建模
  
2010s: 数据仓库现代化
  └── Snowflake、Redshift 云数仓兴起
  └── dbt 提出"分析工程"概念
  
2020s: AI 驱动语义层
  └── LLM 赋能自然语言查询
  └── 语义层 + NL2SQL 成为趋势
```

## 🔬 国外技术方案调研

### 1. Google Looker (LookML)

**核心特点：**
- **语义建模语言** - LookML 声明式建模
- **单一事实来源** - 统一指标定义
- **嵌入式分析** - API 优先设计

**技术架构：**
```
┌─────────────────────────────────────┐
│           Looker 应用层              │
│  - Dashboards  - Explores  - API    │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│            LookML 语义层             │
│  - Views (数据表映射)                │
│  - Explores (关联关系)               │
│  - Measures (指标定义)               │
│  - Dimensions (维度定义)             │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│          数据库适配层                │
│  - BigQuery / Snowflake / Redshift  │
│  - 方言转换 + 查询优化               │
└─────────────────────────────────────┘
```

**LookML 示例：**
```lookml
view: orders {
  sql_table_name: `analytics.orders` ;;
  
  dimension: order_id {
    type: number
    primary_key: yes
  }
  
  dimension: created_date {
    type: date
    sql: ${TABLE}.created_at ;;
  }
  
  measure: total_revenue {
    type: sum
    sql: ${TABLE}.amount ;;
  }
  
  measure: order_count {
    type: count
  }
  
  explore: orders {
    join: users {
      type: left_outer
      sql_on: ${orders.user_id} = ${users.id} ;;
    }
  }
}
```

**优势：**
- ✅ 语义模型版本控制（Git 集成）
- ✅ 强大的权限管理
- ✅ 查询自动生成和优化
- ✅ 成熟的生态系统

**劣势：**
- ❌ 学习曲线陡峭
- ❌ 闭源，绑定 Google 生态
- ❌ 不支持自然语言查询
- ❌ 价格昂贵（$5K+/月）

**适用场景：** 大型企业，有专业数据分析团队

### 2. dbt (data build tool) + dbt Semantic Layer

**核心特点：**
- **代码即文档** - SQL + YAML 定义
- **测试驱动** - 内置数据质量测试
- **开源生态** - 社区活跃

**技术架构：**
```
┌─────────────────────────────────────┐
│         dbt Cloud / CLI             │
│  - 模型开发  - 测试  - 文档生成      │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│       dbt Semantic Layer            │
│  - Metrics (指标定义)               │
│  - Entities (实体定义)              │
│  - Dimensions (维度定义)            │
│  - Saved Queries (预定义查询)        │
