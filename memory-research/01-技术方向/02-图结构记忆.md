## 1. 技术概述

### 1.1 核心原理

图结构记忆使用知识图谱 (节点 + 边) 来表示实体及其关系，支持复杂的关系推理和多跳查询。

```
实体 (节点) --关系 (边)--> 实体 (节点)
   用户  --------喜欢------>  咖啡
   用户  --------居住------>  北京
   北京  --------首都------>  中国
```

### 1.2 与向量记忆对比

| 维度 | 向量记忆 | 图结构记忆 |
|------|----------|------------|
| **检索方式** | 语义相似度 | 关系遍历 |
| **擅长场景** | 模糊查询 | 精确关系 |
| **查询示例** | "我喜欢的饮料" | "我去年见过哪些投资人" |
| **推理能力** | 弱 | 强 (多跳) |
| **可解释性** | 低 | 高 |
| **存储开销** | 低 | 中 |

```python
from neo4j import GraphDatabase
from mem0 import Memory

# 1. 初始化 Neo4j 客户端
class GraphMemory:
    def __init__(self, uri, user, password):
        self.driver = GraphDatabase.driver(uri, auth=(user, password))
    
    def close(self):
        self.driver.close()
    
    def add_entity(self, user_id: str, entity: str, entity_type: str):
        """添加实体节点"""
        with self.driver.session() as session:
            session.run("""
                MERGE (e:Entity {id: $entity_id})
                SET e.name = $entity_name,
                    e.type = $entity_type,
                    e.user_id = $user_id
            """, entity_id=f"{user_id}:{entity}", 
                entity_name=entity, 
                entity_type=entity_type,
                user_id=user_id)
    
    def add_relation(self, user_id: str, from_entity: str, relation: str, to_entity: str):
        """添加关系边"""
        with self.driver.session() as session:
            session.run("""
                MATCH (a:Entity {id: $from_id})
                MATCH (b:Entity {id: $to_id})
                MERGE (a)-[r:RELATION {type: $relation_type}]->(b)
                SET r.user_id = $user_id,
                    r.created_at = timestamp()
            """, from_id=f"{user_id}:{from_entity}",
                to_id=f"{user_id}:{to_entity}",
                relation_type=relation,
                user_id=user_id)
    
    def query_relations(self, user_id: str, from_entity: str = None, relation: str = None):
        """查询关系"""
        with self.driver.session() as session:
            if from_entity and relation:
                result = session.run("""
                    MATCH (a:Entity {id: $from_id})-[r:RELATION {type: $rel_type}]->(b:Entity)
                    WHERE a.user_id = $user_id
                    RETURN b.name as target, r.type as relation
                """, from_id=f"{user_id}:{from_entity}", 
                    rel_type=relation, 
                    user_id=user_id)
            elif from_entity:
                result = session.run("""
                    MATCH (a:Entity {id: $from_id})-[r:RELATION]->(b:Entity)
                    WHERE a.user_id = $user_id
                    RETURN b.name as target, r.type as relation
                """, from_id=f"{user_id}:{from_entity}", user_id=user_id)
            else:
                result = session.run("""
                    MATCH (a:Entity)-[r:RELATION]->(b:Entity)
                    WHERE a.user_id = $user_id OR b.user_id = $user_id
                    RETURN a.name as source, r.type as relation, b.name as target
                """, user_id=user_id)
            
            return [record.data() for record in result]
    
    def multi_hop_query(self, user_id: str, start_entity: str, hops: int = 2):
        """多跳查询"""
        with self.driver.session() as session:
            result = session.run("""
                MATCH path = (a:Entity {id: $start_id})-[:RELATION*1..$hops]-(b:Entity)
                WHERE a.user_id = $user_id
                RETURN [node in nodes(path) | node.name] as entities,
                       [rel in relationships(path) | rel.type] as relations
            """, start_id=f"{user_id}:{start_entity}", hops=hops, user_id=user_id)
            
            return [record.data() for record in result]

# 2. 与 Mem0 集成
class Mem0GraphMemory:
    def __init__(self, mem0_config, neo4j_config):
        self.memory = Memory(**mem0_config)
        self.graph = GraphMemory(**neo4j_config)
    
    def add_memory_with_graph(self, user_id: str, content: str):
        """添加记忆并同时更新图谱"""
        # 1. 添加到向量记忆
        self.memory.add(content, user_id=user_id)
        
        # 2. 抽取实体和关系 (简化版)
        entities = self._extract_entities(content)
        relations = self._extract_relations(content)
        
        # 3. 更新图谱
        for entity, entity_type in entities:
            self.graph.add_entity(user_id, entity, entity_type)
        
        for from_ent, rel, to_ent in relations:
            self.graph.add_relation(user_id, from_ent, rel, to_ent)
    
    def _extract_entities(self, text: str):
        """实体抽取 (简化版，生产环境用 NER 模型)"""
        # 这里只是示例，实际应该用 SpaCy/Stanza
        entities = []
        # 示例规则
        if "喜欢" in text:
            entities.append(("用户", "person"))
        return entities
    
    def _extract_relations(self, text: str):
        """关系抽取 (简化版)"""
        relations = []
        # 示例规则
        if "喜欢" in text:
            relations.append(("用户", "喜欢", "咖啡"))
        return relations

# 使用示例
if __name__ == "__main__":
    # 初始化
    mem0_config = {
        "vector_store": {"provider": "qdrant", "config": {"url": "http://localhost:6333"}}
    }
    neo4j_config = {
        "uri": "bolt://localhost:7687",
        "user": "neo4j",
        "password": "password"
    }
    
    mem_graph = Mem0GraphMemory(mem0_config, neo4j_config)
    
    # 添加记忆
    mem_graph.add_memory_with_graph("user_123", "用户喜欢冰美式咖啡")
    mem_graph.add_memory_with_graph("user_123", "用户居住在北京")
    mem_graph.add_memory_with_graph("user_123", "北京是中国的首都")
    
    # 查询关系
    relations = mem_graph.graph.query_relations("user_123", "用户")
    print("关系:", relations)
    
    # 多跳查询

    print("多跳路径:", paths)
```

### 3.2 Dify 工作流实现

Dify 目前不原生支持图数据库，但可以通过 HTTP 请求节点集成。

#### 步骤 1: 部署图查询 API
```python
# FastAPI 示例
from fastapi import FastAPI
from neo4j import GraphDatabase

app = FastAPI()

driver = GraphDatabase.driver("bolt://localhost:7687", auth=("neo4j", "password"))

@app.post("/graph/query")
async def query_graph(user_id: str, query_type: str, entity: str = None):
    with driver.session() as session:
        if query_type == "relations":
            result = session.run("""
                MATCH (a:Entity {id: $from_id})-[r:RELATION]->(b:Entity)
                WHERE a.user_id = $user_id
                RETURN b.name as target, r.type as relation
            """, from_id=f"{user_id}:{entity}", user_id=user_id)
            return [record.data() for record in result]
        elif query_type == "multi_hop":
            result = session.run("""
                MATCH path = (a:Entity {id: $start_id})-[:RELATION*1..2]-(b:Entity)
                WHERE a.user_id = $user_id
                RETURN [node in nodes(path) | node.name] as entities
            """, start_id=f"{user_id}:{entity}", user_id=user_id)
            return [record.data() for record in result]
```

## 4. 应用场景

### 4.1 复杂关系查询

**场景**: "我去年见过哪些可能对我的 AI 项目感兴趣的投资人？"

**图查询**:
```cypher
MATCH (user:Entity {id: "user_123"})-[:MET {year: 2025}]->(investor:Entity {type: "investor"})
MATCH (investor)-[:INTERESTED_IN]->(field:Entity {name: "AI"})
RETURN investor.name as investor, investor.company as company
```

**向量记忆无法直接回答此类问题**。

### 4.2 知识推理

**场景**: 根据用户记忆推理潜在需求

```
用户 --喜欢--> 咖啡
咖啡 --属于--> 饮品
饮品 --包含--> 茶
─────────────────────
推理：用户可能也喜欢茶
```

### 4.3 冲突检测

**场景**: 检测记忆冲突

```cypher
// 查找矛盾关系
MATCH (user:Entity {id: "user_123"})-[:LIKE]->(item1)
MATCH (user)-[:DISLIKE]->(item2)
WHERE item1.name = item2.name
RETURN item1.name as conflict
```

## 5. 最佳实践

### 5.1 图谱设计原则

```
推荐:
- 节点类型清晰 (人/地点/事物/概念)
- 关系语义明确 (喜欢/居住/工作/认识)
- 添加时间属性 (created_at, updated_at)
- 按用户 ID 隔离数据

避免:
- 节点类型过多 (>20 种)
- 关系类型模糊 (相关/涉及)
- 忽略时间维度
- 多用户数据混存
```

### 5.3 混合架构 (向量 + 图)

```python
class HybridMemory:
    def __init__(self, vector_config, graph_config):
        self.vector_memory = Memory(**vector_config)
        self.graph_memory = GraphMemory(**graph_config)
    
    def search(self, user_id: str, query: str):
        """混合检索"""
        # 1. 向量检索 (模糊查询)
        vector_results = self.vector_memory.search(query, user_id=user_id)
        
        # 2. 图检索 (精确关系)
        entities = self._extract_query_entities(query)
        graph_results = []
        for entity in entities:
            relations = self.graph_memory.query_relations(user_id, entity)
            graph_results.extend(relations)
        
        # 3. 结果融合
        return {
            "vector": vector_results,
            "graph": graph_results
        }
```

## 6. 常见问题

### Q1: 图数据库和向量数据库都要部署吗？
**A**: 取决于需求:
- 只需模糊查询 → 向量数据库即可
- 需要关系推理 → 两者都部署
- 预算有限 → 先用向量，后续扩展

### Q2: 实体抽取不准确怎么办？
**A**:
1. 使用成熟的 NER 模型 (SpaCy/Stanza)
2. 针对领域微调模型
3. 添加人工审核流程

### Q3: 图谱太大查询变慢？
**A**:
1. 按用户 ID 分图
2. 添加索引
3. 限制查询深度
4. 定期清理过期关系
