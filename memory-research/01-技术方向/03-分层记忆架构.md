## 1. 技术概述

### 1.1 核心原理

分层记忆将记忆按重要性和访问频率分为多个层级，类似计算机的存储层次结构 (缓存→内存→硬盘)。

```
┌─────────────────────────────────────────────────────────────┐
│                    分层记忆架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐  ← 快速访问，容量小                    │
│  │  核心记忆       │     (系统提示/用户身份)                │
│  │  (Core Memory)  │                                        │
│  ├─────────────────┤  ← 中等速度，中等容量                  │
│  │  归档记忆       │     (历史对话/文档知识)                │
│  │  (Archival)     │                                        │
│  ├─────────────────┤  ← 慢速访问，容量大                    │
│  │  对话记忆       │     (当前会话上下文)                   │
│  │  (Conversation) │                                        │
│  └─────────────────┘                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 与人脑类比

| 记忆层级 | 人脑类比 | 特点 | 更新频率 |
|----------|----------|------|----------|
| 核心记忆 | 长期记忆/技能 | 稳定、重要 | 低 (周/月) |
| 归档记忆 | 情景记忆 | 可检索、可遗忘 | 中 (天/周) |
| 对话记忆 | 工作记忆 | 临时、快速 | 高 (实时) |

## 2. 核心组件

### 2.1 Letta (原 MemGPT)

**特点**:
- 分层记忆管理的先驱
- 自主记忆整理 (类似操作系统分页)
- 支持技能和子代理

**安装**:
```bash
pip install letta
letta configure
```

### 2.2 LangChain Memory 类型

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| **ConversationBufferMemory** | 完整对话历史 | 短对话 |
| **ConversationSummaryMemory** | 摘要压缩 | 长对话 |
| **ConversationSummaryBufferMemory** | 混合模式 | 通用 |
| **VectorStoreRetrieverMemory** | 向量检索 | 大规模 |
| **KnowledgeGraphMemory** | 知识图谱 | 关系推理 |

### 3.2 Dify 工作流实现

Dify 通过「变量」+「知识库」实现分层记忆。

#### 配置步骤

**步骤 1: 核心记忆 (应用变量)**
```
在应用设置中配置变量:
- user_profile: 用户画像
- preferences: 用户偏好
- goals: 用户目标
```

**步骤 2: 对话记忆 (系统内置)**

Dify 自动管理对话历史，可在应用设置中配置:
- 最大对话轮数：20
- 上下文窗口：8192 tokens

## 4. 记忆整理策略

### 4.1 自动摘要

```python
def summarize_conversation(conversation: List[str], llm) -> str:
    """对话摘要"""
    prompt = f"""
    请总结以下对话的关键信息，包括:
    1. 用户提到的重要事实
    2. 用户的偏好和需求
    3. 达成的共识或决定
    
    对话内容:
    {conversation}
    
    总结 (200 字以内):
    """
    return llm.generate(prompt)
```

### 4.2 时间衰减

```python
def apply_time_decay(memories: List, decay_factor: float = 0.95):
    """应用时间衰减"""
    current_time = datetime.now()
    
    for mem in memories:
        age_days = (current_time - mem["created_at"]).days
        mem["weight"] = mem["score"] * (decay_factor ** age_days)
    
    return sorted(memories, key=lambda x: x["weight"], reverse=True)
```

### 4.3 重要性评分

```python
def calculate_importance(content: str) -> float:
    """计算记忆重要性 (0-1)"""
    score = 0.5  # 基础分
    
    # 规则 1: 包含关键词加分
    important_keywords = ["喜欢", "讨厌", "重要", "决定", "计划"]
    if any(kw in content for kw in important_keywords):
        score += 0.2
    
    # 规则 2: 用户明确要求记住
    if "请记住" in content or "记住我" in content:
        score += 0.3
    
    # 规则 3: 重复提及加分
    # (需要查询历史)
    
    return min(score, 1.0)
```

## 6. 常见问题

### Q1: 三层记忆如何划分边界？
**A**:
- **核心记忆**: 改变频率低 (>1 月)，对用户理解关键
- **归档记忆**: 可检索的历史，改变频率中 (天/周)
- **对话记忆**: 当前会话，改变频率高 (实时)

### Q2: 记忆整理会丢失信息吗？
**A**: 摘要会丢失细节，但保留关键信息。建议:
1. 重要信息立即归档原文
2. 对话摘要保留关键引用
3. 提供用户导出原始对话的选项

### Q3: 如何平衡记忆容量和性能？
**A**:
1. 核心记忆限制在 3000 tokens 以内
2. 对话记忆限制在最近 20-50 轮
3. 归档记忆无上限，但检索时限制 Top-K
