## 1. 方案概述

### 1.1 适用场景

| 场景 | 推荐度 | 说明 |

| 功能 | 描述 | 适用场景 |

## 3. 实施方案

### 3.1 方案 A: 基础版 (仅对话历史)

**适用**: 简单场景，只需短期记忆

**步骤**:
1. 创建「对话型」应用
2. 配置对话历史设置
3. 完成

**配置**:
```yaml
应用类型：对话型
对话历史:
  最大轮数：20
  上下文窗口：8192 tokens
  角色设定：你是用户的个人助理，记住用户的偏好和习惯
```

**优点**:
- ✅ 配置简单 (5 分钟)
- ✅ 零代码
- ✅ 自动管理

**缺点**:
- ❌ 会话结束即遗忘
- ❌ 无法跨会话记忆
- ❌ 无法主动管理记忆

---

### 3.2 方案 B: 标准版 (知识库 + 变量)

**适用**: 需要长期记忆的大多数场景

**步骤 1: 创建知识库**
```
1. 进入「知识库」→「创建知识库」
2. 名称：long_term_memory
3. 索引方式：向量检索
4. Embedding 模型：bge-m3
5. 向量维度：1024
6. 相似度阈值：0.6
```

**步骤 2: 创建应用变量**
```
在应用设置中添加变量:
- user_profile (文本): 用户画像
- preferences (文本): 用户偏好
- goals (文本): 用户目标
```

**步骤 3: 创建工作流**

```

│  └────────┬────────┘                                       │
│           │                                                 │
│           ▼                                                 │
│  ┌─────────────────┐                                       │
│  │  知识库检索     │ ← 从长期记忆检索                       │
│  │  查询：{{message}}                                      │

│  └────────┬────────┘                                       │
```

**步骤 4: 记忆更新 (手动/自动)**

**手动更新**:
```
在知识库中手动添加文档:
- 文档标题：user_123_memories
- 内容：
  用户姓名：张三
  偏好：冰美式咖啡
  居住：北京
  ...
```

**自动更新** (需要 Webhook):
```
配置 Webhook:

- URL: https://your-api.com/memory/update
- 内容：对话摘要
```

### 3.3 方案 C: 进阶版 (HTTP 请求 + 外部记忆服务)

**适用**: 需要完整记忆管理功能

**架构**:
```
Dify 工作流 → HTTP 请求 → 外部记忆服务 (Python API)
                                     ↓
                              Qdrant + SQLite
```

**步骤 1: 部署外部记忆服务**

参考 `01-Python 环境实施方案.md` 部署记忆 API。

**步骤 2: 配置 Dify HTTP 请求节点**

```yaml
节点类型：HTTP 请求
请求方式：POST
URL: http://your-api.com/memories/search

请求头:
  Content-Type: application/json

请求体:
  {
    "user_id": "{{user.id}}",
    "query": "{{message}}",
    "limit": 5
  }

响应解析:
  {{#response.body#}}
```

**步骤 3: 完整工作流**


## 4. 配置详解

### 4.1 知识库配置

**索引设置**:
```yaml
知识库名称：long_term_memory
索引方式：向量检索
Embedding 模型：bge-m3
向量维度：1024
相似度阈值：0.6
Top-K: 5
```

**文档结构**:
```markdown
# 用户记忆：user_123

## 基本信息
- 姓名：张三
- 职业：软件工程师
- 地点：北京

## 偏好
- 咖啡：冰美式
- 饮食：少辣
- 沟通：简洁

## 历史对话摘要
- 2026-03-01: 咨询 AI 学习路径
- 2026-03-05: 讨论项目架构
- 2026-03-10: 询问职业发展
```

### 4.2 变量配置

```yaml
应用变量:
  - 变量名：user_profile
    类型：文本
    默认值：{}
    描述：用户画像 JSON
    
  - 变量名：preferences
    类型：文本
    默认值：{}
    描述：用户偏好 JSON
    
  - 变量名：session_id
    类型：文本
    默认值：{{conversation_id}}
    描述：会话 ID
```

### 4.3 提示词模板

```yaml
系统提示词:
  你是用户的智能助理，拥有以下长期记忆:
  
  === 用户画像 ===
  {{user_profile}}
  
  === 用户偏好 ===
  {{preferences}}
  
  === 相关记忆 ===
  {{#context#}}
  
  请基于以上信息，为用户提供个性化的回复。
  保持回复简洁、有用、友好。
  
  用户消息：{{message}}
```

## 5. 最佳实践

### 5.1 记忆更新策略

**方案 A: 手动更新 (推荐初期)**
```
每周定期:
1. 导出对话历史
2. 人工审核重要信息
3. 更新知识库文档
```

**方案 B: 自动摘要 (推荐中期)**
```
配置 Webhook:
- 每 10 轮对话触发
- 调用 LLM 生成摘要
- 自动添加到知识库
```

**方案 C: 实时提取 (推荐后期)**
```
工作流中添加:
- 意图识别节点
- 信息提取节点
- 条件分支 (是否写入记忆)
- HTTP 请求 (写入记忆服务)
```

### 5.2 隐私保护

```yaml
Dify 隐私设置:
  - 数据加密：启用
  - 日志保留：30 天
  - 数据导出：支持
  - 数据删除：支持
  
敏感信息处理:
  - 不在知识库中存储身份证号、银行卡号
  - 使用变量存储敏感信息 (加密)
  - 定期清理过期记忆
```

### 5.3 性能优化

```yaml
知识库优化:
  - 按用户分库：user_123_memories, user_456_memories
  - 定期重建索引：每月一次
  - 清理低质量文档：相似度 < 0.5

检索优化:
  - Top-K 限制：5-10
  - 相似度阈值：0.6-0.7
  - 启用重排序：是
```

## 8. 常见问题

### Q1: 知识库检索不准确？
**A**:
1. 检查 Embedding 模型是否适合中文
2. 调整相似度阈值 (0.5-0.7)
3. 优化文档结构 (清晰分段)
4. 添加更多相关文档

### Q2: 如何跨会话记忆？
**A**:
1. 使用知识库存储长期记忆
2. 使用变量存储用户 ID
3. 每次对话从知识库检索

### Q3: 如何自动更新记忆？
**A**:
1. 配置 Webhook (对话完成事件)
2. 调用外部 API 生成摘要
3. 自动添加到知识库
4. 或使用 HTTP 请求节点实时提取
