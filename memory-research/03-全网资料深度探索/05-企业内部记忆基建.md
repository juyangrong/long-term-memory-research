# 企业内部记忆基建调研

**最后更新**: 2026-03-22  
**调研范围**: 携程内部记忆相关基础设施  
**资料来源**: 内部文档 + 公开资料

---

## 📋 概述

现阶段公司内部 AI 应用众多，如 TripDify 平台存在众多 AI 应用，默认情况下这些应用之间的上下文是**不互通**的。这意味着你在应用 A 聊天的内容（用户个人信息，如姓名、维护的产品、备忘信息等）无法在应用 B 中得到回应。

**用户画像长期记忆是刚需**，本调研汇总公司内部可用的记忆基建方案。

---

## 🏗️ 方案一：AIS 平台长期记忆 (Memobase)

### 基本信息

| 项目 | 详情 |
|------|------|
| **平台名称** | AIS 长期记忆组件 |
| **技术基础** | Memobase (开源框架) |
| **部署方式** | 公司内部托管 |
| **集成平台** | Dify / TripDify |
| **访问入口** | AIS 平台 - 长期记忆模块 |

### 核心特性

#### 1. 用户绑定机制
- 每个记忆与 `user_id` 绑定
- `user_id` 与用户工号一一绑定
- 同一用户在不同应用间共享记忆
- 突破单会话限制

#### 2. Profiles 自定义
支持自定义用户 Profiles 范畴，例如：

```yaml
基本信息:
  - 年龄
  - 性别
  - 姓名

爱好:
  - 旅游
  - 游戏
  - 烹饪

工作:
  - 维护的产品
  - 工号
  - 岗位职责
```

#### 3. 记忆工具集

| 工具 | 功能 | 调用时机 |
|------|------|----------|
| 创建用户 | 初始化用户记忆空间 | 首次使用 |
| 插入数据 | 写入新的记忆信息 | LLM 后/特定场景 |
| 获取用户上下文 | 读取用户记忆数据 | 每次 LLM 之前 |
| 刷新记忆缓冲区 | 立即写入长期记忆 | 需要持久化时 |

### 使用流程

#### 步骤 1: 创建项目
在 AIS 平台创建 Memobase 项目，获取：
- `HOST` - 记忆服务器地址
- `API_KEY` - 认证密钥

#### 步骤 2: 注册 Dify 插件
在 Dify 平台注册 Memobase 插件服务：
- 方式 A: 安装《memobase》Dify 插件（推荐）
- 方式 B: 使用 HTTP 节点请求接口（无插件权限时）

#### 步骤 3: 编排 Dify 应用
在 Dify 应用中添加 Memobase 工具节点：

```
开始 → 获取用户上下文 → LLM → 插入数据 → [刷新缓冲区] → 回复
                        ↑              ↓
                        └──── 上下文 ──┘
```

#### 步骤 4: 测试验证
1. 用户在 TP 群或服务号与机器人交互
2. 告知机器人记忆信息：`"我维护的产品名称为：Trip，appid=37"`
3. 信息传给 Dify 应用，通过 Memobase 节点存储
4. 下次询问时，通过《获取用户上下文》节点获取记忆
5. 验证 LLM 能正确响应上下文信息

### 注意事项

| 序号 | 要求 | 说明 |
|------|------|------|
| 1 | 每次 LLM 前获取上下文 | 确保 LLM 携带用户记忆 |
| 2 | 每次 LLM 后或特定场景插入数据 | 保存新的记忆信息 |
| 3 | 需要时手动刷新缓冲区 | 立即写入长期记忆 |
| 4 | 缓存自动写入 | 缓存满或超时后自动持久化 |

### API 详细说明

#### 1. add_memory - 添加记忆

**功能**: 从用户指令中识别偏好，保存到记忆服务

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `content` | string | ✅ | 用户输入的原始消息内容 |
| `user_id` | string | ❌ | 用户身份 id，为空时从 headers/query 获取 |
| `run_id` | string | ❌ | 会话 id，为空时从 headers/query 获取 |
| `agent_id` | string | ❌ | 应用 id，为空时从 headers/query 获取 |
| `metadata` | string | ❌ | JSON 对象字符串，记忆数据的属性特征 |

**metadata 示例**:
```json
// 爱好分类
{"category":"hobbies"}

// Coding 工程信息
{"project":"my_project","language":"python","framework":"flask","database":"mysql","cloud_service":"aliyun"}

// 未来计划行为
{"plan":"travel_to_shanghai","date":"2025-12-05","destination":"shanghai"}

// 重要个人信息
{"name":"张三","age":"20","gender":"male","email":"zhangsan@example.com","phone":"12345678901"}
```

#### 2. search_memory - 检索记忆

**功能**: 按用户输入内容检索最相关的记忆数据

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `content` | string | ✅ | 用户输入的原始消息内容 |
| `user_id` | string | ❌ | 用户身份 id |
| `run_id` | string | ❌ | 会话 id |
| `agent_id` | string | ❌ | 应用 id |
| `limit` | string | ❌ | 返回条数，默认 100 |
| `filters` | string | ❌ | JSON 对象字符串，过滤条件 |

**filters 示例**:
```json
// 工作场景
{"scene":"work"}

// 特定项目
{"project":"my_project"}

// 计划行为
{"plan":"travel_to_shanghai"}

// 个人信息
{"name":"张三"}
```

#### 3. update_memory - 更新记忆

**功能**: 根据 memory_id 更新指定记忆内容

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `memory_id` | string | ✅ | 记忆数据 ID |
| `content` | string | ✅ | 新的偏好内容文本 |

#### 4. get_memory - 获取单条记忆

**功能**: 根据 memory_id 获取指定记忆内容

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `memory_id` | string | ✅ | 记忆数据 ID |

#### 5. get_all_memory - 获取所有记忆

**功能**: 获取当前用户的所有记忆内容

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_id` | string | ❌ | 用户身份 id |
| `run_id` | string | ❌ | 会话 id |
| `agent_id` | string | ❌ | 应用 id |
| `limit` | string | ❌ | 返回条数，默认 100 |
| `filters` | string | ❌ | JSON 对象字符串，过滤条件 |

#### 6. delete_memory - 删除记忆

**功能**: 根据 memory_id 删除指定记忆内容

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `memory_id` | string | ✅ | 记忆数据 ID |

### 使用示例

#### 示例 1: 添加用户偏好
```python
import requests

url = "http://aigw.fx.ctripcorp.com/mcp/100000013/mcp"

# 添加用户偏好
response = requests.post(url, json={
    "method": "add_memory",
    "params": {
        "content": "用户喜欢冰美式咖啡",
        "user_id": "employee_123",
        "metadata": '{"category":"hobbies","type":"beverage"}'
    }
})
```

#### 示例 2: 检索记忆
```python
# 检索咖啡相关偏好
response = requests.post(url, json={
    "method": "search_memory",
    "params": {
        "content": "咖啡",
        "user_id": "employee_123",
        "limit": "10",
        "filters": '{"category":"hobbies"}'
    }
})

# 返回最相关的记忆数据
```

#### 示例 3: 获取所有工作相关记忆
```python
# 获取工作场景下的所有记忆
response = requests.post(url, json={
    "method": "get_all_memory",
    "params": {
        "user_id": "employee_123",
        "filters": '{"scene":"work"}'
    }
})
```

#### 示例 4: 更新记忆
```python
# 更新指定记忆内容
response = requests.post(url, json={
    "method": "update_memory",
    "params": {
        "memory_id": "mem_abc123",
        "content": "用户喜欢热拿铁咖啡 (更新)"
    }
})
```
