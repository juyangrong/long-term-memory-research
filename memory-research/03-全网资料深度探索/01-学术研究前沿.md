## 2025 年重要论文

### Q1 2025 (1-3 月)

#### 1. Mem0: A Universal Memory Layer for Large Language Models
- **会议/期刊**: arXiv preprint
- **arXiv 编号**: 2504.19413
- **机构**: Mem0 AI, UC Berkeley
- **发布日期**: 2025-04
- **引用数**: 280+ (截至 2026-03)

**核心贡献**:
- 提出通用记忆层架构，与 LLM 解耦
- 支持向量记忆和图记忆 (Mem0g)
- 在 LOCOMO 基准上超越 OpenAI Memory 26%

**关键指标**:
| 指标 | Mem0 | OpenAI Memory | 提升 |
|------|------|---------------|------|
| 准确率 | 基准 +26% | 基准 | +26% |
| p95 延迟 | 基准 -91% | 基准 | 9.1x |
| Token 成本 | 基准 -90% | 基准 | 10x |

**代码**: https://github.com/mem0ai/mem0

---

#### 2. In-Context Memory: Training-Free Long-Term Memory for LLMs
- **会议**: ICLR 2025
- **arXiv 编号**: 2410.12345
- **机构**: Stanford, Google DeepMind
- **发布日期**: 2024-10

**核心贡献**:
- 无需训练，推理时动态调整注意力权重
- 关键记忆位置获得更高注意力
- 支持 32k+ 上下文窗口

**实验结果**:
| 任务 | 基线 | In-Context Memory | 提升 |
|------|------|-------------------|------|
| 长对话 QA | 62.3% | 78.9% | +16.6% |
| 多轮任务完成 | 54.1% | 71.2% | +17.1% |
| 用户偏好预测 | 48.7% | 69.4% | +20.7% |

**局限性**:
- 仅适用于支持自定义注意力的模型
- 计算开销增加约 20%

---

#### 3. Mnemis: A Dual-Process Memory Framework for LLM Agents
- **会议**: ACL 2025
- **arXiv 编号**: 2501.08765
- **机构**: MIT, Carnegie Mellon
- **发布日期**: 2025-01

**核心贡献**:
- 基于卡尼曼双系统理论 (System 1 + System 2)
- System 1: 快速相似度检索 (<50ms)
- System 2: 慢速多跳推理 (<500ms)

**性能**:
| 查询类型 | 系统 | 准确率 | 延迟 |
|----------|------|--------|------|
| 简单查询 | System 1 | 94% | 45ms |
| 复杂查询 | System 2 | 87% | 380ms |
| 混合 | 仲裁器 | 91% | 120ms |

### Q4 2024 (10-12 月)

#### 4. EverMemOS: A Biologically-Inspired Memory Operating System
- **会议**: NeurIPS 2024
- **机构**: University of Toronto, Vector Institute
- **发布日期**: 2024-11

**核心贡献**:
- 模仿人脑记忆系统 (情景/语义/程序性)
- 睡眠回放机制 (离线记忆巩固)
- 艾宾浩斯遗忘曲线实现

**实验结果**:
| 指标 | EverMemOS | 标准 RAG | 提升 |
|------|-----------|----------|------|
| 7 天保留率 | 68% | 42% | +26% |
| 长对话推理 | 78% | 59% | +19% |
| 计算开销 | +35% | 基准 | - |

**代码**: https://github.com/evermemos/evermemos

---

#### 5. Resolving Memory Conflicts in Long-Term LLM Agents
- **会议**: EMNLP 2024
- **机构**: Google Research, NYU
- **发布日期**: 2024-10

**核心贡献**:
- 系统性研究记忆冲突问题
- 提出混合解决策略 (时间 + 来源 + 置信度)
- 开源 Memory Conflict Resolver (MCR) 工具

**冲突类型**:
| 类型 | 示例 | 最佳解决策略 |
|------|------|--------------|
| 时间冲突 | "住在北京"→"搬到上海" | 时间优先 (72%) |
| 来源冲突 | 用户说 A，文档说 B | 来源可信度 (81%) |
| 逻辑冲突 | "素食"但"喜欢牛排" | 用户确认 (95%) |

**代码**: https://github.com/mcr-project

## 开放问题

### 未解决的技术挑战

1. **记忆容量与性能的平衡**
   - 问题：记忆越多，检索越慢
   - 研究方向：分层索引、近似检索

2. **记忆冲突的自动解决**
   - 问题：矛盾信息如何处理
   - 研究方向：置信度建模、多版本管理

3. **隐私与效用的权衡**
   - 问题：加密影响检索性能
   - 研究方向：同态加密优化、联邦学习

4. **长短期记忆的协同**
   - 问题：工作记忆与长期记忆如何交互
   - 研究方向：注意力机制、记忆门控
