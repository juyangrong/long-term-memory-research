### Mem0

**GitHub**: https://github.com/mem0ai/mem0  
**文档**: https://docs.mem0.ai/  
**版本**: v1.0.0 (2025-04)

**核心特性**:
- ✅ 向量记忆 + 图记忆 (Mem0g)
- ✅ 多用户支持
- ✅ 敏感信息过滤
- ✅ GDPR 导出/删除
- ✅ 云托管/自托管/混合

**技术栈**:
```yaml
向量库：Qdrant, Pinecone, Chroma
嵌入：OpenAI, HuggingFace, Cohere
LLM: OpenAI, Anthropic, Ollama, 本地
部署：Docker, Kubernetes, 云托管
```

**快速开始**:
```python
from mem0 import Memory

memory = Memory(
    vector_store={"provider": "qdrant", "config": {"url": "http://localhost:6333"}},
    llm={"provider": "ollama", "config": {"model": "llama3.1:8b"}}
)

memory.add("用户喜欢冰美式咖啡", user_id="user_123")
results = memory.search("咖啡偏好", user_id="user_123")
```

**性能基准** (LOCOMO):
| 指标 | Mem0 | OpenAI Memory |
|------|------|---------------|
| 准确率 | +26% | 基准 |
| p95 延迟 | -91% | 基准 |
| Token 成本 | -90% | 基准 |

**优点**:
- ✅ 性能优异，基准领先
- ✅ 文档完善，易于上手
- ✅ 活跃社区，快速迭代
- ✅ 多种部署选项

**缺点**:
- ⚠️ 图记忆功能仍在测试
- ⚠️ 云托管成本较高

**适用场景**:
- ✅ 通用 AI 助手
- ✅ 客服系统
- ✅ 个人助理

### Letta (原 MemGPT)

**GitHub**: https://github.com/cpacker/Letta  
**文档**: https://docs.letta.com/  
**版本**: v0.5 (2025-02)

**核心特性**:
- ✅ 分层记忆 (核心/归档/对话)
- ✅ 自主记忆整理
- ✅ 技能系统
- ✅ 多代理协作
- ✅ Web UI

**技术栈**:
```yaml
向量库：Chroma, Qdrant, PostgreSQL
LLM: OpenAI, Anthropic, Ollama
部署：Docker, 云托管
```

**快速开始**:
```python
from letta import create_client

client = create_client()
agent = client.create_agent(
    name="assistant",
    memory_type="hierarchical",
    core_memory={
        "human": "用户名称：张三",
        "persona": "你是专业助手"
    }
)

response = client.send_message(agent.id, "你好", role="user")
```

**优点**:
- ✅ 分层记忆先驱
- ✅ 自主记忆整理
- ✅ 完善的 Web UI
- ✅ 多代理支持

**缺点**:
- ⚠️ 学习曲线陡峭
- ⚠️ 资源消耗较大
- ⚠️ 文档相对复杂

**适用场景**:
- ✅ 复杂代理系统
- ✅ 需要自主记忆管理
- ✅ 研究探索

### LangChain Memory

**GitHub**: https://github.com/langchain-ai/langchain  
**文档**: https://python.langchain.com/docs/modules/memory/  
**版本**: 持续更新

**核心特性**:
- ✅ 多种记忆类型
- ✅ 与 LangChain 生态深度集成
- ✅ 灵活定制
- ✅ 活跃社区

**记忆类型**:
| 类型 | 描述 | 适用场景 |
|------|------|----------|
| ConversationBufferMemory | 完整对话历史 | 短对话 |
| ConversationSummaryMemory | 摘要压缩 | 长对话 |
| VectorStoreRetrieverMemory | 向量检索 | 大规模 |
| KnowledgeGraphMemory | 知识图谱 | 关系推理 |
| ConversationSummaryBufferMemory | 混合模式 | 通用 |

**快速开始**:
```python
from langchain.memory import ConversationSummaryBufferMemory

memory = ConversationSummaryBufferMemory(
    llm=llm,
    max_token_limit=2000,
    memory_key="chat_history"
)
```

**优点**:
- ✅ 生态丰富，工具众多
- ✅ 灵活定制
- ✅ 文档完善
- ✅ 社区活跃

**缺点**:
- ⚠️ 记忆功能相对基础
- ⚠️ 需要自行整合

**适用场景**:
- ✅ LangChain 用户
- ✅ 需要灵活定制
- ✅ 复杂工作流

### Memobase

**GitHub**: https://github.com/memobase/memobase  
**文档**: https://memobase.readthedocs.io/  
**版本**: v0.3+ (2024-11)

**核心特性**:
- ✅ 结构化记忆 (Profiles)
- ✅ 用户绑定机制
- ✅ 缓存 + 持久化双层存储
- ✅ 多工具集成 (Dify/HTTP)
- ✅ 工号绑定 (企业场景)

**技术栈**:
```yaml
存储：PostgreSQL + 缓存层
API: RESTful / HTTP
部署：Docker, 自托管
集成：Dify 插件，HTTP 节点
```

**快速开始**:
```python
from memobase import MemobaseClient

client = MemobaseClient(
    host="http://localhost:8080",
    api_key="your_api_key"
)

# 创建用户
client.create_user(user_id="employee_123")

# 插入记忆
client.insert_memory(
    user_id="employee_123",
    data={"产品": "携程 App", "appid": "99999999"}
)

# 获取上下文
context = client.get_user_context(user_id="employee_123")

# 刷新缓冲区 (立即持久化)
client.refresh_buffer(user_id="employee_123")
```

**记忆结构示例**:
```yaml
Profiles:
  基本信息:
    - 姓名
    - 年龄
    - 性别
  爱好:
    - 旅游
    - 游戏
    - 烹饪
  工作:
    - 维护的产品
    - 工号
    - 岗位职责
```

**优点**:
- ✅ 结构化 Profiles，易于管理
- ✅ 缓存优化，减少频繁写入
- ✅ 企业场景友好 (工号绑定)
- ✅ Dify 插件化集成
- ✅ 轻量级，学习成本低

**缺点**:
- ⚠️ 社区相对较小
- ⚠️ 需要手动管理刷新时机
- ⚠️ 依赖 Dify 应用编排

**适用场景**:
- ✅ 企业 AI 应用
- ✅ 需要结构化记忆
- ✅ Dify/TripDify 平台
- ✅ 跨应用用户画像共享

**企业案例**:
- ✅ 携程集团：AIS 平台长期记忆组件
- 使用规模：内部 AI 应用 + 员工
- 集成方式：Dify 插件 + MCP 服务双轨制
