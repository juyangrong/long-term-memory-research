# 定时任务配置说明

_配置日期：2026-03-20_

---

## 📋 配置概览

### 1. Heartbeat 任务 (HEARTBEAT.md)

**位置：** `HEARTBEAT.md`

**检查频率：** 由主会话的 heartbeat 轮询触发

**任务内容：**
- 每日检查：整理记忆日志、查看未回复消息
- 每周检查：整理上周日志到长期记忆
- 每月检查：审查并更新 MEMORY.md

---

### 2. 系统 Cron 任务

**脚本位置：** `~/.openclaw/scripts/heartbeat.sh`

**执行时间：**
- 每天 09:00 - 早间检查
- 每天 20:00 - 晚间检查

**日志位置：** `~/.openclaw/logs/heartbeat.log`

---

### 3. 状态追踪

**状态文件：** `memory/heartbeat-state.json`

记录：
- 上次检查时间
- 配置参数
- 保留周期（30 天）

---

## 🔧 管理命令

```bash
# 查看当前 cron 配置
crontab -l

# 编辑 cron 配置
crontab -e

# 删除所有 cron 任务
crontab -r

# 手动执行 heartbeat 脚本
~/.openclaw/scripts/heartbeat.sh

# 查看 heartbeat 日志
tail -f ~/.openclaw/logs/heartbeat.log
```

---

## 📝 注意事项

1. **脚本路径：** 确保 `~/.openclaw/scripts/heartbeat.sh` 存在且有执行权限
2. **日志目录：** 首次运行时会自动创建 `~/.openclaw/logs/`
3. **时区：** 系统时区应为 Asia/Shanghai
4. **OpenClaw 状态：** 脚本会检查 OpenClaw 是否运行，未运行时跳过

---

## 🔄 记忆整理流程

```
每日会话 → memory/YYYY-MM-DD.md (短期记忆)
              ↓
    (Heartbeat 定期检查)
              ↓
    提炼重要内容 (4-5 分)
              ↓
MEMORY.md + 专题文件 (长期记忆)
```

**重要性评分：**
- 5 分：核心知识 → MEMORY.md
- 4 分：重要信息 → 专题文件
- 2-3 分：日常记录 → 每日日志（30 天衰减）
- <2 分：临时信息 → 不记录
