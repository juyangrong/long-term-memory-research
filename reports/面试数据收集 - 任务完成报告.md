## 🚀 如何使用

### 步骤 1: 查看候选人简历

```bash
# 打开摩卡系统
https://app.mokahr.com/interviews/overview?period=future

# 查看每个候选人的详细信息
# 记录关键信息到文档
```

### 步骤 2: 填写基本信息

```markdown
**基本信息：**
- 姓名：张三
- 应聘职位：大数据开发工程师
- 工作年限：5 年
- 当前公司：某互联网公司
- 学历：本科
- 面试时间：2026-03-18 14:00
```

### 步骤 3: 选择面试问题

根据候选人技术栈，从文档中选择合适的问题：

**如果候选人会 Spark：**
```
1. Spark Shuffle 过程如何优化？
2. 数据倾斜如何解决？
3. Spark 内存管理模型？
```

**如果候选人会 Flink：**
```
1. Flink Checkpoint 原理？
2. Watermark 的作用？
3. 反压机制如何工作？
```

### 步骤 4: 面试评分

面试过程中：
1. 记录每个维度的得分
2. 记录关键观察
3. 给出录用建议

### 步骤 5: 对比分析

所有候选人面试完成后：
1. 填写对比表
2. 确定优先级
3. 给出最终建议

## 📊 面试问题示例

### 技术基础问题

**Java：**
- HashMap 和 ConcurrentHashMap 的区别？
- 线程池的核心参数有哪些？
- JVM 内存模型是怎样的？

**Spark：**
- Spark 执行流程？
- Shuffle 过程如何优化？
- 数据倾斜解决方案？

**Flink：**
- Checkpoint 原理？
- Watermark 的作用？
- 如何保证 Exactly Once？

**Kafka：**
- 如何保证消息不丢失？
- 分区策略有哪些？
- Consumer Group 原理？

### 项目经验深挖

**STAR 法则追问：**
- **S**ituation: 项目背景是什么？
- **T**ask: 你负责的任务是什么？
- **A**ction: 你采取了什么行动？
- **R**esult: 结果如何？（数据指标）

**深入问题：**
- 项目中遇到的最大技术挑战？
- 如何解决的？
- 如果重新做，会怎么改进？
- 有什么遗憾或不足？

### 系统设计题

**初级：**
- 设计一个短链接系统
- 设计一个计数器

**中级：**
- 设计一个实时数据仓库
- 设计一个消息队列

**高级：**
- 设计一个亿级数据量的推荐系统
- 设计一个数据中台
