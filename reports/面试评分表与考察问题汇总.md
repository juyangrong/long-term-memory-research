## 2. 技术能力考察问题

### 2.1 Java 基础

#### 初级问题
1. Java 中的 HashMap 和 ConcurrentHashMap 有什么区别？
2. 谈谈你对 Java 内存模型的理解
3. synchronized 和 ReentrantLock 的区别是什么？

#### 中级问题
1. JVM 垃圾回收机制有哪些？G1 回收器的特点是什么？
2. 如何排查和解决 Java 内存泄漏问题？
3. 谈谈你对 Java 并发包 (JUC) 的理解和使用经验

#### 高级问题
1. 设计一个高并发的缓存系统，需要考虑哪些问题？
2. 如何优化 JVM 参数以提升系统性能？
3. 谈谈你对虚拟线程 (Project Loom) 的理解和看法

---

### 2.2 数据库 (MySQL)

#### 初级问题
1. 数据库索引的原理是什么？什么情况下索引会失效？
2. 事务的 ACID 特性是什么？
3. 什么是脏读、幻读、不可重复读？

#### 中级问题
1. 如何优化慢查询？请分享你的优化经验
2. MySQL 的锁机制有哪些？如何避免死锁？
3. 分库分表的策略有哪些？什么时候需要考虑分库分表？

#### 高级问题
1. 设计一个支持海量数据的订单表，你会如何设计？
2. MySQL 主从延迟的原因和解决方案有哪些？
3. 如何保证数据库和缓存的数据一致性？

---

### 2.3 大数据技术 (Spark/Flink)

#### Spark 问题

**初级：**
1. Spark 的 RDD、DataFrame、Dataset 有什么区别？
2. 什么是宽依赖和窄依赖？对性能有什么影响？
3. Spark 的 shuffle 过程是怎样的？

**中级：**
1. 如何优化 Spark 任务的性能？请分享你的优化经验
2. Spark 内存管理模型是怎样的？
3. 如何处理数据倾斜问题？

**高级：**
1. 设计一个实时数据处理 pipeline，如何选择 Spark Streaming 还是 Structured Streaming？
2. Spark on K8s 的架构和优势是什么？
3. 如何保证 Spark 任务的Exactly-Once语义？

#### Flink 问题

**初级：**
1. Flink 的时间语义有哪些？Event Time 和 Processing Time 的区别？
2. 什么是 Checkpoint 和 Savepoint？
3. Flink 的状态后端有哪些？

**中级：**
1. Flink 如何处理反压问题？
2. 窗口计算有哪些类型？如何使用？
3. Flink CEP 的使用场景是什么？

**高级：**
1. 设计一个实时风控系统，如何使用 Flink 实现？
2. Flink 的 Exactly-Once 是如何保证的？
3. Flink 1.14+ 的新特性有哪些？

---

### 2.4 消息队列 (Kafka)

#### 初级问题
1. Kafka 的架构是怎样的？Producer、Consumer、Broker 的关系？
2. 什么是 Topic、Partition、Offset？
3. Kafka 如何保证消息的顺序性？

#### 中级问题
1. Kafka 如何保证消息不丢失？
2. Consumer Group 的工作原理是什么？
3. 如何优化 Kafka 的吞吐量？

#### 高级问题
1. 设计一个支持亿级消息量的消息系统，如何选型和设计？
2. Kafka 的事务机制是怎样的？
3. 如何处理消息积压问题？

### 2.5 系统设计

#### 场景题

**1. 实时数据平台建设**
> 设计一个支持每日百亿数据量的实时数据平台

**考察点：**
- 数据采集 (Flume/Kafka)
- 实时计算 (Flink/Spark)
- 数据存储 (ClickHouse/Doris)
- 数据服务 (API 网关)
- 监控告警

**2. 用户画像系统**
> 设计一个支持千万用户实时标签计算的系统

**考察点：**
- 标签体系设计
- 实时计算架构
- 存储选型 (HBase/ES)
- 查询优化
- 更新策略

**3. 推荐系统数据链路**
> 设计推荐系统的数据处理和特征工程链路

**考察点：**
- 特征存储
- 实时特征计算
- 样本生成
- 数据质量
- A/B 测试支持

## 3. 项目经历考察问题

### 3.1 项目深度挖掘

#### STAR 法则提问

**Situation (情境):**
1. 请介绍一下这个项目的背景
2. 项目要解决的核心问题是什么？
3. 当时的技术选型考虑是什么？

**Task (任务):**
1. 你在项目中负责的具体工作是什么？
2. 你面临的主要挑战是什么？
3. 项目的目标是什么？

**Action (行动):**
1. 你是如何解决这些挑战的？
2. 做了哪些技术决策？为什么？
3. 与团队是如何协作的？

**Result (结果):**
1. 项目最终的结果如何？
2. 有哪些量化的成果？
3. 有什么可以改进的地方？

---

### 3.2 技术细节追问

#### 架构设计
1. 为什么选择这个架构？有什么权衡？
2. 系统的瓶颈在哪里？如何优化？
3. 如果数据量增长 10 倍，架构需要如何调整？

#### 技术难点
1. 项目中遇到的最大技术难点是什么？
2. 你是如何解决的？
3. 有没有其他解决方案？

#### 性能优化
1. 做过哪些性能优化？
2. 优化前后的对比数据？
3. 优化的思路和方法论是什么？

---

### 3.3 项目匹配度评估

| 维度 | 问题 | 评价 |
|------|------|------|

## 7. 面试流程指南

### 7.1 面试前准备

**面试官准备：**
- [ ] 阅读候选人简历
- [ ] 准备针对性问题
- [ ] 准备 coding 题目
- [ ] 准备系统设计题
- [ ] 熟悉评分标准

**环境准备：**
- [ ] 预定会议室
- [ ] 测试视频设备
- [ ] 准备白板/纸笔
- [ ] 准备面试评分表

---

### 7.2 面试流程

```
┌─────────────────────────────────────────────────────────┐
│                    面试流程 (90 分钟)                     │
├─────────────────────────────────────────────────────────┤
│  0-5 分钟   │  开场介绍，缓解紧张情绪                    │
│  5-25 分钟  │  项目经历深挖 (STAR 法则)                   │
│  25-50 分钟 │  技术能力考察 (基础知识 + 技术深度)          │
│  50-70 分钟 │  系统设计/coding 测试                      │
│  70-85 分钟 │  软技能和文化匹配考察                      │
│  85-90 分钟 │  候选人提问，面试总结                      │
└─────────────────────────────────────────────────────────┘
```

---

### 7.3 面试后跟进

**面试官：**
- [ ] 完成面试评分表
- [ ] 撰写面试反馈
- [ ] 提交录用建议
- [ ] 参与面试讨论

**HR：**
- [ ] 收集面试反馈
- [ ] 安排后续面试
- [ ] 跟进录用流程
- [ ] 候选人沟通

---

### 7.4 注意事项

**Do's (应该做):**
- ✅ 营造轻松的面试氛围
- ✅ 多问开放性问题
- ✅ 给候选人思考时间
- ✅ 做好面试记录
- ✅ 及时反馈

**Don'ts (不应该做):**
- ❌ 打断候选人回答
- ❌ 问歧视性问题
- ❌ 过早暴露倾向
- ❌ 面试中处理其他事务
- ❌ 拖延反馈时间
